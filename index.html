Fundamentos

Protocolos em Camadas

  • Devido à ausência de memória compartilhada, toda comunicação em sistemas distribuídos é baseada no envio e recebimento de mensagens.
  • Quando um processo A quer se comunicar com um processo B, primeiro ele deve montar uma mensagem em seu próprio espaço de endereço.
  • Para enviar a mensagem, o processo A faz uma chamada de sistema (syscall) solicitando ao SO que envie a mensagem pela rede até B.

    • OSI (Open Systems Interconnection Reference Model)
      • Foi criado para padronizar um conjunto de regras para a comunicação entre sistemas
    • Essas regras são denominadas protocolos.
    • Se um grupo de computadores quiserem se comunicar, devem concordar em utilizar um protocolo.
  • Vários acordos são necessários para que a comunicação tenha sucesso e responder questões como:
    • Existem dois tipos básicos de protocolos:
      • Protocolos Orientados a Conexão: antes de trocar dados, os comunicantes devem estabelecer uma conexão. Ex: TCP
      • Protocolos sem Conexão: o remetente apenas transmite a primeira mensagem quando estiver pronta. Ex: UDP

    • No entanto, os protocolos desenvolvidos pela Internet, como o TCP/IP, são mais populares.

Fundamentos

Tipos de comunicação

  • Middleware:
    • Comunicação Persistente: a mensagem é armazenada pelo middleware durante o tempo que for necessário.
    • Comunicação Transiente: a mensagem é armazenada pelo middleware enquanto as aplicações comunicantes estiverem executando.
  • Aplicação:
    • Comunicação Assíncrona: o remetente continua executando após enviar a mensagem.
    • Comunicação Síncrona: o remetente fica bloqueado após enviar a mensagem, até receber uma resposta.

RPC

Chamada de Procedimento Remoto

  • Programas podem chamar procedimentos localizados em outra máquinas.
  • Um processo na máquina A chama um pede para um processo B realizar um procedimento.
  • O processo A fica bloqueado.
  • A execução do procedimento ocorre em B.
  • B retorna para A o resultado da execução.

Comunicação orientada a Mensagem

  • A comunicação por RPC adota a premissa de que o receptor está apto para atender a requisições.
  • Transiente: terminal de comunicação de rede.
    • Um Socket é vinculado a um IP e um número de Porta
    • O servidor passa a ouvir de um lado do socket, aceitando as mensagens que chegam no IP e Porta definidos
    • O cliente usa o endereço do servidor para fazer uma conexão e enviar dados.
  • Persistente: Middleware Orientado a Mensagem (MOM)
    • É um sistema de fila de mensagens
    • Não exige que o remetente ou o receptor estejam ativos durante a transmissão
    • O middleware age como gerenciador da fila de mensagens
    • Filas e Publish/Subscribe
